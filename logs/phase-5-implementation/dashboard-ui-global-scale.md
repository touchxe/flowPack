# 대시보드 전역 UI 스케일 개선

## 날짜

- 2026-07-07

## 배경

- 홈 화면 개선 기준을 왼쪽 메뉴와 주요 회원용 페이지 전체에 확장했다.
- KRDS의 글자/화면 표시 설정, 컴포넌트, 기본 패턴 분류를 참고해 국내형 업무 대시보드에서 익숙한 큰 글자, 명확한 카드, 여유 있는 필터/표 행 밀도를 우선 적용했다.

## 적용 범위

- 공통 레이아웃: 앱 사이드바, 상단바, 본문 최대 폭, 페이지 헤더, 상태 카드, 빈 상태, 로딩 스피너
- 주요 페이지: 통계, 캘린더, 콘텐츠 목록, 미디어, SNS 연동, 수정피드, 작성 지침, 대량 기획, URL 콘텐츠 변환, 카드뉴스 생성, 블로그 초안 작성, 콘텐츠 편집, 설정 하위 화면

## 주요 변경

- 사이드바 폭을 넓히고 메뉴 행, 아이콘, 사용자 영역 크기를 상향했다.
- 본문 기본 여백과 최대 폭을 키워 KPI 카드와 테이블이 답답하지 않도록 조정했다.
- 주요 버튼/필터/검색 입력/선택 컨트롤 높이를 40-58px 범위로 정리했다.
- KPI 카드, 차트 카드, 목록 행, 편집 패널 패딩을 키워 홈 화면과 같은 시각 밀도로 맞췄다.
- 기존 `any` 단언 1건을 제거하고 크레딧 사용률 계산의 0 나누기 가능성을 방어했다.

## 검증

- `node node_modules/typescript/bin/tsc --noEmit`
- `npm run build`
