# [Phase 1] 작업 로그 — 인터뷰 에이전트

## 기본 정보
- **날짜**: 2026-03-31
- **Phase**: Phase 1: 인터뷰 에이전트
- **소요 시간**: 약 1시간 (사용자 요구사항 수신 및 문서화)
- **관련 Task**: PHASE-1

## 이번에 한 일
사용자로부터 FlowPack 프로젝트의 요구사항을 수신했다. AI 기반 홍보 콘텐츠 생성·배포 플랫폼으로, 스타트업·개인사업자·소호 사업자를 타겟으로 한다. 핵심 기능은 AI 콘텐츠 생성(카드뉴스·블로그·영상), SNS/블로그 원클릭 배포, 통계 대시보드, 티어별 요금제다. 요구사항을 `docs/requirements-main.md`와 `docs/prd.md`로 구조화하고, 5개의 엣지 케이스를 추가로 정의했다.

## 핵심 결정 사항
| 결정 | 선택지 | 선택 이유 |
|------|--------|---------|
| 프로젝트명 | FlowPack | 사용자 제공 |
| MVP 범위 | 인증·생성·배포·통계·결제 | 1차 개발 확정 |
| 2차 제외 | Homepage 제작 기능 | 복잡도 관리, 추후 추가 |
| 소셜 로그인 | Google·Kakao·Apple | 사용자 제안, 추가 확인 필요 |

## 사용한 도구/기술
- 요구사항 → `docs/requirements-main.md` (원본 정리)
- 요구사항 → `docs/prd.md` (PRD 형식으로 구조화 + 사용자 스토리 + 엣지 케이스 추가)

## 어려웠던 점 / 배운 점
- 사용자가 이미 정리된 요구사항을 제공해 인터뷰 4라운드 절차 없이 빠르게 Phase 1 완료 가능
- 미확정 항목 3개 (무료 티어 수량 상한, 요금 금액, 소셜 로그인 제공사 최종 확정) → Phase 3 아키텍처 설계 전에 반드시 확정 필요

## AI와의 협업 노트
- 사용자 제공 요구사항을 그대로 받아 prd.md 에 사용자 스토리 형식(Epic별 US-XXX)으로 변환
- 엣지 케이스는 Phase 1 문서에 없던 내용을 추가 보완
