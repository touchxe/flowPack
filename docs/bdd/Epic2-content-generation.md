## US-010: AI 홍보글 생성 (기본)

### Feature: AI 홍보글 생성 - 파라미터 선택

```gherkin
Feature: AI 홍보글 생성 - 파라미터 선택
  Scenario: 사용자가 모든 필수 파라미터를 선택하여 콘텐츠를 생성한다

    Given 로그인한 사용자가 "/carousel-lab" 페이지에 접속한다
    When 사용자가 업종으로 "카페"를 선택한다
    And 톤으로 "친근한"을 선택한다
    And 스타일로 "홍보성"을 선택한다
    And 길이로 "중간(5-7슬라이드)"을 선택한다
    And 주제로 "봄 시즌 한정 음료 출시"를 입력한다
    And "생성하기" 버튼을 클릭한다
    Then 로딩 상태가 표시된다
    And SSE 스트리밍으로 진행 메시지가 표시된다
    When 생성 완료 시
    Then 생성된 카드뉴스 미리보기가 표시된다
    And "임시 저장" 메시지가 표시된다
    And 크레딧 1건이 차감된다
    And "/carousel-lab" 페이지에 결과가 표시된다
```

```gherkin
  Scenario: 필수 파라미터가 누락된 상태로 생성 시도

    Given 로그인한 사용자가 "/carousel-lab" 페이지에 접속한다
    When 사용자가 업종으로 "카페"만 선택한다
    And 주제를 입력하지 않는다
    And "생성하기" 버튼을 클릭한다
    Then 에러 메시지 "주제를 입력해주세요"가 표시된다
    And 콘텐츠가 생성되지 않는다
    And 크레딧이 차감되지 않는다
```

```gherkin
  Scenario: 크레딧이 모두 소진된 경우

    Given 로그인한 사용자의 크레딧이 0이다
    When 사용자가 "/carousel-lab" 페이지에 접속한다
    And 파라미터를 모두 입력한다
    And "생성하기" 버튼을 클릭한다
    Then "크레딧이 부족합니다" 게이팅 모달이 표시된다
    And 콘텐츠가 생성되지 않는다
    And "요금제 업그레이드하기" 버튼이 표시된다
```

## US-011: 생성된 글 직접 편집

### Feature: 콘텐츠 편집

```gherkin
Feature: 콘텐츠 편집
  Scenario: 사용자가 생성된 카드뉴스의 슬라이드 내용을 수정한다

    Given 로그인한 사용자가 생성된 카드뉴스를 보유하고 있다
    When 사용자가 해당 카드뉴스의 "편집" 버튼을 클릭한다
    Then 편집 모드로 전환된다
    When 사용자가 첫 번째 슬라이드 텍스트를 "새로운 제목"으로 변경한다
    And 변경 사항을 저장한다
    Then "저장 완료" 토스트 메시지가 표시된다
    And 변경 사항이 반영된다
```

```gherkin
  Scenario: 사용자가 카드뉴스에 슬라이드를 추가한다

    Given 로그인한 사용자가 카드뉴스 편집 모드이다
    When 사용자가 "슬라이드 추가" 버튼을 클릭한다
    Then 새로운 빈 슬라이드가末尾に追加된다
    When 사용자가 해당 슬라이드에 내용을 입력한다
    And 저장한다
    Then 슬라이드가 추가된 콘텐츠가 저장된다
```

```gherkin
  Scenario: 사용자가 생성된 블로그 포스트의 본문을 수정한다

    Given 로그인한 사용자가 생성된 블로그 포스트를 보유하고 있다
    When 사용자가 해당 블로그의 "편집" 버튼을 클릭한다
    Then 편집 에디터가 표시된다
    When 사용자가 본문 내용을 수정한다
    And "저장" 버튼을 클릭한다
    Then 수정 사항이 저장된다
```

## US-012: AI 이미지 생성

### Feature: AI 이미지 생성

```gherkin
Feature: AI 이미지 생성
  Scenario: 사용자가 카드뉴스에 사용할 이미지를 생성한다

    Given 로그인한 사용자가 카드뉴스 편집 모드이다
    And 첫 번째 슬라이드를 선택했다
    When 사용자가 "AI 이미지 생성" 버튼을 클릭한다
    Then 이미지 생성 팝업이 표시된다
    When 프롬프트에 "봄 꽃과 커피가 있는 카페风光"를 입력한다
    And 스타일로 "실사"를 선택한다
    And 화면 비율로 "1:1"을 선택한다
    And "생성" 버튼을 클릭한다
    Then 로딩 상태가 표시된다
    When 생성 완료 시
    Then 생성된 이미지 미리보기가 표시된다
    When 사용자가 "적용" 버튼을 클릭한다
    Then 이미지가 슬라이드에 삽입된다
```

```gherkin
  Scenario: 이미지 생성 중 오류 발생

    Given 로그인한 사용자가 이미지 생성 팝업을 열고 있다
    When AI 이미지 생성 중 오류가 발생한다
    Then 에러 메시지 "이미지 생성에 실패했습니다. 다시 시도해주세요."가 표시된다
    And 크레딧이 차감되지 않는다
```

## US-013: URL을 입력해 콘텐츠 변환

### Feature: URL → 콘텐츠 변환

```gherkin
Feature: URL 입력하여 콘텐츠 변환
  Scenario: 사용자가 유효한 URL을 입력하여 콘텐츠를 생성한다

    Given 로그인한 사용자가 "/ai/bulk-link-to-post" 페이지에 접속한다
    When URL 입력창에 "https://news.example.com/article/123"을 입력한다
    And "변환하기" 버튼을 클릭한다
    Then 로딩 상태가 표시된다
    When URL 콘텐츠 분석 완료 시
    Then 카드뉴스 미리보기가 표시된다
    And 원본 URL 정보가 함께 표시된다
```

```gherkin
  Scenario: 유효하지 않은 URL 입력

    Given 로그인한 사용자가 "/ai/bulk-link-to-post" 페이지에 접속한다
    When URL 입력창에 "not-a-valid-url"을 입력한다
    And "변환하기" 버튼을 클릭한다
    Then 에러 메시지 "유효한 URL을 입력해주세요"가 표시된다
```

## US-014: 카드뉴스(Carousel) 형식 생성


```gherkin
  Scenario: 생성된 카드뉴스를 PNG로 내보낸다
```

## US-015: 장문 블로그 포스트 AI 생성

### Feature: 장문 블로그 생성

```gherkin
Feature: 장문 블로그 생성

    When 사용자가 주제 "인스타그램 마케팅의 모든 것"을 입력한다
    And 키워드로 ["인스타그램", "마케팅", "바이럴"]을 입력한다
    And 길이로 "긴（约1500단어）"를 선택한다
    And 톤으로 "전문적인"을 선택한다
    And "생성하기" 버튼을 클릭한다

    When 생성 완료 시
    Then 마크다운 형식의 블로그 포스트가 표시된다
    And 글자 수가 표시된다
```

```gherkin
  Scenario: 생성 중 사용자가 취소를 클릭한다

    Given 로그인한 사용자가 긴 블로그 포스트 생성 중이다
    When SSE 스트리밍中进行中に 사용자가 "취소" 버튼을 클릭한다
    Then 생성이 중단된다
    And 부분적으로 생성된 콘텐츠가 초안으로 저장된다
    And 크레딧이 일부만 차감된다
```

## US-016: 대량 기획 생성

### Feature: 대량 콘텐츠 기획

```gherkin
Feature: 대량 콘텐츠 기획
  Scenario: 사용자가 여러 주제의 콘텐츠를 한 번에 생성策划한다

    Given 로그인한 사용자가 "/ai/bulk-generate" 페이지에 접속한다
    When 사용자가 주제 목록에 다음을 입력한다
      | 주제 | 유형 | 슬라이드 수 |
      | 봄 시즌 한정 메뉴 | carousel | 5 |
      | 신규 매장 오픈 소식 | blog | - |
      | 고객 후기 모음 | carousel | 8 |
    And "일괄 생성" 버튼을 클릭한다
    Then 3개의 콘텐츠가 큐에 추가된다
    And 각 콘텐츠별로 생성 상태가 표시된다
    When 모든 생성이 완료되면
    Then 완료된 콘텐츠 목록이 표시된다
    And 총 크레딧 사용량이 표시된다
```

## 엣지 케이스

```gherkin
  Scenario: AI 생성 API 타임아웃

    Given 로그인한 사용자가 콘텐츠 생성 요청을 했다
    When OpenAI API가 60초 이내에 응답하지 않는다
    Then 재시도 로직이 작동한다（최대 3회）
    When 3회 재시도 후에도 실패하면
    Then "생성에 실패했습니다" 에러 메시지가 표시된다
    And 크레딧이 차감되지 않는다
    And 사용자에게 "잠시 후 다시 시도해주세요" 안내가 표시된다
```

```gherkin
  Scenario: 생성된 콘텐츠의 모든 슬라이드가 비어있는 경우

    Given AI가 생성한 콘텐츠의 모든 슬라이드가 비어있다
    Then "생성된 콘텐츠가 비어있습니다. 다시 생성해주세요." 안내가 표시된다
    And 크레딧이 환불된다
```

```gherkin
  Scenario: 사용자가 생성된 콘텐츠를 삭제한다

    Given 로그인한 사용자가 보유한 초안 콘텐츠가 있다
    When 사용자가 해당 콘텐츠의 "삭제" 버튼을 클릭한다
    Then 확인 다이얼로그가 표시된다
    When 사용자가 "삭제"를 확인한다
    Then 콘텐츠가 ARCHIVED 상태로 변경된다
    And 목록에서 숨겨진다
```
