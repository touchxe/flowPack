## US-030: 포스팅별 조회수 확인

### Feature: 대시보드 통계 확인

```gherkin
Feature: 대시보드 통계
  Scenario: 사용자가 대시보드에서 전체 통계를 확인한다

    Given 로그인한 사용자가 "/home" 페이지에 접속한다
    Then 다음 통계 카드가 표시된다
      | 지표 | 현재값 |
      | 이번 달 생성 | 3 |
      | 총 조회수 | 1,240 |
      | 배포 완료 | 7 |
    And "전월 대비" trend 값이 표시된다
```

```gherkin
  Scenario: 사용자가 상세 통계 페이지에서 조회수를 확인한다

    Given 로그인한 사용자가 "/analytics" 페이지에 접속한다
    Then 기간 선택 드롭다운이 표시된다（7일/30일/90일）
    When 사용자가 "30일"을 선택한다
    Then 지난 30일간의 통계가 표시된다
    And 그래프로 추이가 시각화된다
```

### Feature: 콘텐츠별 상세 통계

```gherkin
  Scenario: 사용자가 특정 콘텐츠의 상세 통계를 확인한다

    Given 로그인한 사용자가 "/analytics" 페이지에 접속한다
    When 사용자가 특정 콘텐츠 항목을 클릭한다
    Then 해당 콘텐츠의 상세 통계 모달이 표시된다
    And 플랫폼별 성능이 나열된다
      | 플랫폼 | 조회수 | 좋아요 | 공유 |
      | Instagram | 450 | 32 | 5 |
      | Facebook | 280 | 18 | 2 |
      | Twitter | 120 | 8 | 1 |
    And 일별 추이 그래프가 표시된다
```

## US-031: 콘텐츠 캘린더에서 예약·관리

### Feature: 콘텐츠 캘린더 보기

```gherkin
Feature: 콘텐츠 캘린더
  Scenario: 사용자가 월간 캘린더视图을 본다

    Given 로그인한 사용자가 "/calendar" 페이지에 접속한다
    Then 현재 월의 캘린더가 표시된다
    And 예약된 콘텐츠가 해당 날짜에 표시된다
    And 각 예약 항목에 콘텐츠 제목과 상태 배지가 표시된다
```

```gherkin
  Scenario: 사용자가 주간 뷰로 전환한다

    Given 로그인한 사용자가 "/calendar" 페이지에 접속한다
    When 사용자가 "주간" 뷰 버튼을 클릭한다
    Then 주간 캘린더로 전환된다
    And 시간대별 예약 콘텐츠가 표시된다
```

### Feature: 예약된 콘텐츠 편집

```gherkin
  Scenario: 사용자가 예약된 콘텐츠의 날짜를 변경한다

    Given 사용자가 "4월 1일 14:00"에 예약된 콘텐츠를 보유하고 있다
    When 사용자가 해당 예약 항목을 드래그한다
    Then 날짜/시간 변경 모달이 표시된다
    When 사용자가 "4월 3일 10:00"으로 변경한다
    And "변경" 버튼을 클릭한다
    Then 예약 시각이 업데이트된다
    And "예약이 변경되었습니다" 메시지가 표시된다
```

```gherkin
  Scenario: 사용자가 예약된 콘텐츠를 취소한다

    Given 사용자가 예약된 콘텐츠를 보유하고 있다
    When 사용자가 예약 항목의 "취소" 버튼을 클릭한다
    Then 확인 다이얼로그가 표시된다
    When 사용자가 "취소"를 확인한다
    Then 예약이 삭제된다
    And 상태가 "초안"으로 변경된다
    And 크레딧이 환불된다（이미 차감된 경우）
```

### Feature: 캘린더에서 즉시 배포

```gherkin
  Scenario: 사용자가 캘린더에서 즉시 배포를 실행한다

    Given 사용자가 초안 상태의 콘텐츠를 보유하고 있다
    When 사용자가 "/calendar" 페이지에서 해당 콘텐츠를 선택한다
    And "지금 배포" 버튼을 클릭한다
    Then 배포 채널 선택 모달이 표시된다
    When 채널을 선택하고 배포를 시작한다
    Then 배포가 즉시 시작된다
```

## US-032: 콘텐츠 목록 관리

### Feature: 콘텐츠 필터링 및 검색

```gherkin
  Scenario: 사용자가 콘텐츠 유형으로 필터링한다

    Given 로그인한 사용자가 "/home" 또는 콘텐츠 목록 페이지에 접속한다
    When 사용자가 필터에서 "카드뉴스"를 선택한다
    Then 카드뉴스 유형의 콘텐츠만 표시된다
    And "전체" 선택 시 모든 유형이 표시된다
```

```gherkin
  Scenario: 사용자가 키워드로 콘텐츠를 검색한다

    Given 로그인한 사용자가 콘텐츠 목록 페이지에 접속한다
    When 검색창에 "봄"을 입력한다
    Then 제목이나 본문에 "봄"이 포함된 콘텐츠만 표시된다
    And 실시간으로 검색 결과가 갱신된다
```

## US-033: 콘텐츠 아카이브

### Gherkin
```gherkin
  Scenario: 사용자가 오래된 콘텐츠를 아카이브한다

    Given 로그인한 사용자가 보유한 콘텐츠 중 "2025년 12월" 이전的作品が複数存在する
    When 사용자가 해당 콘텐츠들을 선택한다
    And "아카이브" 버튼을 클릭한다
    Then 선택된 콘텐츠들이 ARCHIVED 상태로 변경된다
    And 목록에서 숨겨진다
    And "아카이브됨" 필터에서 확인할 수 있다
```

```gherkin
  Scenario: 통계 API 로딩 중 로딩 스피너가 표시된다

    Given 로그인한 사용자가 "/analytics" 페이지에 접속한다
    When 통계를 불러오는 중이면

    And "통계를 불러오는 중..." 텍스트가 보인다
```

```gherkin
  Scenario: 통계 데이터가 없는 경우

    Given 로그인한 사용자가 아직 배포한 콘텐츠가 없다
    When 사용자가 "/analytics" 페이지에 접속한다
    Then " 아직 통계 데이터가 없습니다" 빈 상태 메시지가 표시된다
    And "첫 번째 콘텐츠를 배포해보세요" 안내가 표시된다
```

```gherkin
  Scenario: 캘린더에 예약이 많은 경우（Pagination/Lazy Loading）

    Given 사용자의 예약 콘텐츠가 50건 이상이다
    When 사용자가 "/calendar" 페이지에 접속한다
    Then 처음 20건만 표시된다
    And 스크롤하면 나머지 예약이_lazy loading된다
```
