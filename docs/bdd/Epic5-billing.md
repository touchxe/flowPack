## US-040: 무료 티어 사용

### Feature: 무료 티어 크레딧 정책

```gherkin
Feature: 무료 티어 사용
  Scenario: 새 사용자가 무료 티어로 시작한다

    Given 새 사용자가 회원가입을 완료했다
    Then 사용자의 플랜이 "FREE"로 설정된다
    And 초기 크레딧이 10건으로 부여된다
    And "残り 10회" 크레딧 잔액이 표시된다
```

```gherkin
  Scenario: 무료 티어 사용자가 월간 크레딧이 리셋된다

    Given 무료 티어 사용자가 이번 달 크레딧 10건을 모두 사용했다
    When 다음 달 1일이 되면
    Then 크레딧이 10건으로 리셋된다
    And "크레딧이 리셋되었습니다" 알림이 발송된다
```

### Feature: 크레딧 잔액 표시

```gherkin
  Scenario: 사용자가 크레딧 잔액을 확인한다

    Given 로그인한 사용자가 "/home" 또는 상단 네비게이션에 접속한다
    Then 크레딧 잔액 "3/10건"이 표시된다
    When 사용자가 크레딧 잔액을 클릭한다
    Then 크레딧 상세 팝오버가 표시된다
    And 사용 내역이 나열된다
```

## US-041: 월간/연간 요금제로 업그레이드

### Feature: 요금제 확인

```gherkin
Feature: 요금제 보기
  Scenario: 사용자가 요금제 페이지를 확인한다

    Given 로그인한 사용자가 "/pricing" 페이지에 접속한다
    Then 다음 요금제가 표시된다
      | 요금제 | 월 요금 | 크레딧 | 주요 기능 |
      | FREE | 무료 | 10/월 | 기본 기능 |
      | STARTER | ₩19,900 | 50/월 | 카드뉴스 + 블로그 |
      | PRO | ₩49,900 | 200/월 | 모든 기능 + 우선 지원 |
      | ENTERPRISE | ₩99,900 | 무제한 | 맞춤 개발 + CSM |
    And 연간 선택 시 할인율이 표시된다（약 17% 할인）
```

### Feature: 월간 요금제 구독

```gherkin
  Scenario: 사용자가 월간 PRO 요금제로 업그레이드한다

    Given 로그인한 사용자가 FREE 플랜이다
    And "/pricing" 페이지에 접속했다
    When 사용자가 PRO 요금제의 "월간" "구독하기" 버튼을 클릭한다
    Then Toss Payments checkout 페이지로 이동한다
    When 사용자가 결제 정보를 입력하고 결제한다
    And 결제 성공 시
    Then "/settings/billing" 페이지로 리다이렉트된다
    And "구독이 완료되었습니다" 메시지가 표시된다
    And 플랜이 "PRO"로 업데이트된다
    And 크레딧 한도가 200으로 변경된다
```

```gherkin
  Scenario: 연간 요금제 구독 시 할인 적용

    Given 로그인한 사용자가 PRO 연간 구독을 선택했다
    When 결제 페이지에 표시된 금액을 확인한다
    Then 월간 비용 × 12에서 약 17% 할인이 적용된 금액이 표시된다
    When 결제가 완료되면
    Then 연간 구독이生效한다
```

### Feature: 결제 실패 처리

```gherkin
  Scenario: 결제 중 오류가 발생한다

    Given 사용자가 PRO 요금제 결제를 진행 중이다
    When 카드 정보 오류로 결제가 실패한다
    Then "결제에 실패했습니다. 카드 정보를 확인해주세요." 메시지가 표시된다
    And 구독이 변경되지 않는다
    And 사용자가 다시 시도할 수 있다
```

```gherkin
  Scenario: 카드 한도 초과로 결제 실패

    Given 사용자의 카드가 한도 초과이다
    When 사용자가 결제를 시도한다
    Then "카드 한도가 부족합니다" 메시지가 표시된다
    And 다른 결제 수단 안내가 표시된다
```

## US-042: 구독 취소

### Feature: 구독 취소

```gherkin
  Scenario: 사용자가 월간 구독을 취소한다

    Given PRO 플랜 사용자가 "/settings/billing" 페이지에 접속한다
    When 사용자가 "구독 취소" 버튼을 클릭한다
    Then 구독 취소 확인 모달이 표시된다
    And "취소 시 이번 달 말까지 이용 가능합니다" 안내가 표시된다
    When 사용자가 "취소 확인"을 클릭한다
    Then 구독이 취소된다
    And 플랜이 FREE로 변경되지 않는다（即時ではなく 말일에 변경）
    And "구독이 취소되었습니다. {현재 구독 종료일}까지 이용 가능합니다." 안내가 표시된다
```

## US-044: 결제 수단 관리

### Feature: 결제 수단 등록/삭제

```gherkin
  Scenario: 사용자가 결제 수단을 등록한다

    Given 로그인한 사용자가 "/settings/billing" 페이지에 접속한다
    When 사용자가 "결제 수단 추가" 버튼을 클릭한다
    Then Toss Payments 카드 등록 페이지가 표시된다
    When 사용자가 카드 정보를 입력한다
    And 등록을 완료한다
    Then 결제 수단 목록에 새 카드가 추가된다
```

```gherkin
  Scenario: 사용자가 결제 수단을 삭제한다

    Given 로그인한 사용자가 등록된 결제 수단을 보유하고 있다
    When 사용자가 해당 결제 수단의 "삭제" 버튼을 클릭한다
    Then 확인 다이얼로그가 표시된다
    When 사용자가 "삭제"를 확인한다
    Then 결제 수단이 목록에서 제거된다
```

## US-045: 크레딧 부족 알림

### Feature: 크레딧 부족 시 알림

```gherkin
  Scenario: 크레딧이 2건 이하로 남으면 알림을 받는다

    Given 사용자의 크레딧이 2건이다
    When 사용어가 콘텐츠를 생성한다
    Then "크레딧이 2건 남았습니다. 업그레이드를 고려해보세요." 안내가 표시된다
    And 알림 설정에 따라 이메일/카카오톡으로 알려진다
```

```gherkin
  Scenario: 크레딧이 모두 소진되면 업그레이드 게이팅이 표시된다

    Given 사용자의 크레딧이 0건이다
    When 사용자가 "/carousel-lab" 페이지에 접속한다
    And 파라미터를 입력하고 "생성하기"을 클릭한다
    Then 크레딧 부족 게이팅 모달이 표시된다
    And "크레딧이 모두 소진되었습니다" 안내가 표시된다
    And "STARTER로 업그레이드（₩19,900/月）" 버튼이 표시된다
    And "현재 FREE 플랜 유지" 링크도 표시된다
```

## 엣지 케이스

```gherkin
  Scenario: 이미 PRO 플랜인 사용자가 PRO 구독을 시도한다

    Given PRO 플랜 사용자가 "/pricing" 페이지에 접속한다
    Then PRO 요금제에 "현재 구독 중" 배지가 표시된다
    And "구독하기" 버튼 대신 "관리" 버튼이 표시된다
```

```gherkin
  Scenario: 연간 구독 중 월간으로 변경 요청

    Given 연간 PRO 구독 중이다
    When 사용자가 월간 PRO로 변경을 요청한다
    Then "연간 구독 중에는 월간으로 변경할 수 없습니다. 연간 구독 종료 후 변경됩니다." 안내가 표시된다
```

```gherkin
  Scenario: 결제 웹훅이 중복으로 수신된다

    Given Toss Payments에서 중복 웹훅이 수신된다
    Then 이미 처리된 결제건은 무시된다
    And 결제 건수가 중복으로 처리되지 않는다
```

```gherkin
  Scenario: 크레딧 사용량이 플랜 한도를 초과하려고 시도한다

    Given FREE 플랜 사용자의 크레딧이 10이고 사용하려고 하는 양이 15이다
    When 사용자가 "일괄 생성"으로 15건을 생성하려 하면
    Then "크레딧이 부족합니다（残り 10건）" 에러가 표시된다
    And 생성 가능한 10건까지만 처리된다
```
