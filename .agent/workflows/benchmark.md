# 🔍 Phase 1.5: 벤치마킹

## 트리거
Phase 1(/interview) 완료 후. 사용자가 참조 URL을 제공하거나 "벤치마킹", "참고 사이트", "이런 느낌으로" 등을 언급할 때.

---

## Step 1: 참조 사이트 수집
사용자에게 질문:
- "참고할 서비스/사이트 URL이 있나요? (최대 5개)"
- "어떤 측면을 참고하고 싶나요?" (디자인 / 기능 / UX흐름 / 비즈니스모델)

## Step 2: 사이트 분석 (URL별)
각 URL에 대해 조사:
1. **사이트맵**: 주요 페이지, 네비게이션, 정보 아키텍처
2. **핵심 기능**: 기능 목록, 작동 방식
3. **UX 패턴**: 온보딩, CTA 배치, 사용자 여정
4. **디자인 언어**: 색상, 타이포, 레이아웃, 여백, 컴포넌트 스타일
5. **기술 단서**: 사용 기술 (가능한 범위)
6. **차별점/약점**: 우리가 개선할 포인트

## Step 3: 벤치마킹 보고서 → `docs/benchmarking/benchmark-report.md`
```markdown
# 벤치마킹 보고서

## 참조 사이트 요약
| # | 사이트 | URL | 참고 측면 | 핵심 인사이트 |
|---|--------|-----|----------|--------------| 

## 사이트맵 비교
[각 사이트의 페이지 구조 정리]

## 채택/불채택 결정
| 요소 | 출처 | 채택 | 사유 |
|------|------|------|------|

## 우리 서비스에 적용할 인사이트
[종합 분석]
```

## Step 4: 사이트맵 초안 → `docs/sitemap.md`
벤치마킹 + PRD를 종합하여 도출:
```markdown
# 사이트맵

## 공개 페이지 (비로그인)
- / — 랜딩 페이지
- /pricing — 요금제
- /login — 로그인
- /register — 회원가입

## 인증 후 페이지
- /home — 대시보드 (메인)
- /content — 콘텐츠 관리
- /settings — 설정

## 페이지별 우선순위
| 페이지 | 우선순위 | Epic |
|--------|---------|------|
```

사용자 확인 후 확정.

## Step 5: 작업 로그
`/work-log`로 `logs/phase-1.5-benchmarking.md` 작성.

## 금지 사항
- 참조 사이트의 디자인을 그대로 복사하지 마라.
- 벤치마킹 없이 Phase 2(/wireframe)로 넘어가지 마라 (사용자가 명시적으로 건너뛰기를 요청한 경우 제외).
